Sass is a powerful CSS extension language that is interpreted into CSS
Adds mechanics that are typically available in traditional programming languages
Designed by Hampton Catlin
Developed by Weizenbaum and Eppstein
First appeared in 2006
Current Release: Selective Steve (3.4.21)

Variables created using “$” e.g. $width: 5em

Only available within level of nested selector they are defined, unless “!global” is used

If defined outside of any selectors, they’re available everywhere

Hyphens and underscores are interchangeable

Variables and Data Types

Strings, Lists, Numbers, Booleans, Nulls, Colors, Maps

Colors

p {
   color: #010203 + #040506;
}

Computes 01+04 = 05, 02+05 = 07, 03+06 = 09

Compiles to ⇒
p {
   color: #050709;
}


Sass contains a large list of predefined functions

These can be found at: http://sass-lang.com/documentation/Sass/Script/Functions.html

Functions

RGB Functions
   rgb($red, $green, $blue)
   Creates a color from red, green, blue values
   mix($color1, $color2, [$weight])
String Functions
   unquote($string)
   Add quotes to a string
   str-index($string, $substring)
Number Functions
   random([$limit])
   min($numbers)
   max($numbers)
Sass supports basic control statements

if statements, for loops, while loops, etc.


For Loops

   The format from <start> through <end> specifies
   that the for loop includes <end>

   The format from <start> to <end> goes up to but
   not including <end>

   If <start> is greater than <end> the for loop
   decrements instead of increments

Control Directives

$type: monster;
p {
   @if $type == ocean {
     color: blue;
   } @else if $type == monster {
     color: green;
   } @else {
     color: black;
   }
}
This is compiled to
p {
   color: green;
}
Can nest rules in each other

Inner rule only applies within the outer rule’s selector

Use “&” to reference parent when nesting

Nesting

$border: 1px solid #EFC6F3;
.post {
   border-radius: 3px;
   background: #FFF8FF;
   border: $border;
   padding: 15px;
   color: #333333;
   .title {
     color: #000000;
     font-size: 20px;
   }
   .alt-title {
     @extend .title;
     border-bottom: $border;
   }
}
Sass supports all CSS3 @-rules as well as Sass-specific ones

@ rules are instructions to the CSS parser

@charset, @import, @media, @page, @font-face, @namespace

@ Rules

Same functionality as regular verisons
Capability of nesting is a Sass feature
example.scss
.example {
   color: red;
}
main.scss
#main {
   @import “example”;
}
This compiles to
#main .example {
   color: red;
}
.sidebar {
   width 300px;
   @media screen and (orientation:landscape) {
     width: 500px;
   }
}


Creating reusable chunks of CSS

Avoids repetitive code

Defined with the @mixin directive

Can take arguments, written in a paragraph and separated by a commarr

Mixins

@mixin headline ($color, $size) {
   color: $color;
   font-size: $size;
}
h1 {
   @include headline(green, 12px);

}
h1 {
   color: green;
   font-size: 12px;
}
